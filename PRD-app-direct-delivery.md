# 开启应用推广（直投） PRD

## Problem Statement

当前计划基本信息中，"推广对象"仅开放「网站推广」，无法切换到「应用推广」。投放策略为「直投」的推广链接无法被选用，导致运营无法通过本系统创建应用类投放任务。

## Solution

开放「应用推广」入口，补充监测链接模版配置项，添加推广链接选择侧边栏，完成应用直投场景的配置闭环。

## User Stories

1. 作为投放运营，我想在计划基本信息中切换推广对象为「应用推广」，使后续所有配置围绕应用直投场景展开
2. 作为投放运营，切换为应用推广后，我想看到「监测链接模版」配置项，并能在预览区上方按账户独立配置应用
3. 作为投放运营，我想通过监测链接模版的 [选择] 按钮打开推广链接列表，按投放策略筛选后选中一条并回填到表单
4. 作为投放运营，我想在预览区上方为每个账户单独选择 Kwai 应用，配置方式与发布身份配置一致

## Implementation

### 计划基本信息 — 推广对象

- 推广对象的「应用推广」radio 按钮从灰显禁用改为正常可选
- 切换推广对象时联动显隐：
  - **应用推广**：显示预览区上方「应用选择配置」按钮 + 计划抽屉内「监测链接模版」行
  - **网站推广**：显示「落地页链接」+「Kwai Pixel」（广告组抽屉内），隐藏应用选择配置按钮

### 应用选择配置（按账户）

- 位置：批量抽屉预览区上方，与「发布身份配置」按钮并列，仅在推广对象=应用推广时可见
- 交互：点击「应用选择配置」按钮弹出弹窗
- 弹窗布局：表格（账户 | 应用下拉选择），底部 [取消] [确定]
- 未配置的账户默认使用 netshort (123456)
- 确认后保存到 `accountAppConfig`，提示"应用选择配置已保存"并刷新预览表
- 计划卡片摘要中显示「应用：按账户配置」

### 计划基本信息 — 监测链接模版

- 位置：计划抽屉内，应用推广时显示
- 结构：标签「监测链接模版」→ 只读输入框（placeholder: 请选择推广链接）→ [选择] 按钮
- 交互：点击 [选择] 拉起「选择推广链接」侧边栏

### 侧边栏 — 选择推广链接

- 宽度：屏幕 35%（约 1/3）
- 布局：
  - 顶部：[新增推广链接] 按钮 + 关键字搜索输入框 + [搜索] [重置] 按钮
  - 中部：表格（radio 单选 | ID | 推广链接名称 | 投放策略 | 语言 | 修改/复制/删除）
  - 底部：分页器（共 N 条 + 每页条数 + 页码）+ [取消] [确定]
- 交互：
  - 表格内 radio 单选，打开时默认选中上次确认的值
  - 搜索/重置为前端过滤
  - 点击 [确定] 时校验：

### 投放策略与推广对象校验

| 推广对象 | 可选投放策略 | 不匹配时的处理 |
|---------|-------------|---------------|
| 应用推广 | 直投 | 弹警告："投放策略为「W2A」的链接要求推广对象为「网站推广」，请先在计划基本信息中切换推广对象" |
| 网站推广 | W2A | 弹警告："投放策略为「直投」的链接要求推广对象为「应用推广」，请先在计划基本信息中切换推广对象" |

- 校验通过后关闭侧边栏，将推广链接名称回填到计划基本信息中的「监测链接模版」字段

### 保存计划 — 卡片摘要

- 保存计划时，当推广对象为应用推广，卡片摘要中展示：
  - 应用：按账户配置
  - 监测链接模版：选中的推广链接名称

## Out of Scope

- 推广链接的新增/修改/复制/删除操作（按钮已保留，功能后续迭代）
- 监测链接模版的编辑功能（当前只读，通过侧边栏选择填充）
- 计划重保存时监测链接模版的自动清空逻辑（视为计划级配置，不清空）
