# Kwai 批量创编系统 — 需求文档
## 系统概述
Kwai 批量创编系统是用于批量创建和管理 Kwai（快手海外）广告投放任务的 Web 工具。系统包含两大模块：
- **模块一：批量任务列表** — 展示、查询和管理批量投放任务，监控计划/广告组/广告三层级提交状态
- **模块二：配置区** — 创建新投放任务，配置账号、计划、推广链接、广告组、定向包、创意等参数，最后提交生成批量任务
**页面入口**: 投放系统 > Kwai投放
---
# 模块一：批量任务列表
## 1.1 模块概述
批量任务列表是系统的默认主页面，展示每一批次中各层级（计划 → 广告组 → 广告）的提交状态和结果，支持筛选、重试、删除等操作。

- 输入框使用 `.el-input__inner` 类，带边框和圆角

### 1.3.4 结果列
- 格式: `成功:N 失败:N`

- 点击重试后: 该层级状态重置为"等待提交"，结果重置为 `成功:0 失败:0`
- 重试按钮位于结果列，不在状态列

## 1.7 数据流
### 1.7.1 任务生命周期
创建任务记录（planStatus="等待提交"）→ Kwai API 批量调用 → 计划提交/广告组提交/广告提交 → 成功/失败 → 各层级 status="提交完成" → [重试] 失败项恢复为"等待提交"
### 1.7.2 重试机制
- 仅对**单个层级**（计划/广告组/广告）的失败项重试
- 重试后该层级状态立即变为"等待提交"，结果清零
- 不影响其他层级和其他任务

# 模块二：配置区
## 2.1 模块概述
配置区是一个**全屏抽屉**，由主页面 [批量添加] 按钮或表格行 [详情] 按钮触发打开。用户在此配置投放任务所需的账户、计划、推广链接、广告组、定向包、创意等参数，最后提交生成批量任务。
**进入方式**:
- 点击模块一工具栏的 [批量添加] 按钮 → 新建模式
- 点击模块一表格行操作列的 [详情] 按钮 → 查看/编辑模式
## 2.2 抽屉结构
- 遮罩层 (Overlay) z-index 动态递增，起始 2000，`openDrawer` 每次 +2
- 抽屉 (Drawer) z-index = Overlay + 1，width: 100%
- Header: 配置区 + [选择模板] [保存模板] [预览项目]
- 卡片滚动区: 横向滚动，5 张卡片（账号 / 计划基本信息 / 广告组 / 定向包 / 广告）
- 预览区: 双行表头表格 + 预计生成统计 + [提交任务] [定时提交]
- 每个卡片点击底部操作按钮打开对应配置子抽屉；定向包卡片通过弹窗选择
- 配置区 header 右侧三个按钮：选择模板、保存模板、预览项目
- 预览区 header 右侧两个按钮：提交任务、定时提交
---
## 2.3 表单控件规范
### 2.3.1 Radio Button Tabs（选项卡切换）
所有配置抽屉中的选择类字段统一使用 radio button tabs 代替下拉框，样式特征：
- 使用 `.el-radio-group` + `.el-radio-button--small` 组合
- 选中态：蓝色背景 (`--el-color-primary`)，白色文字
- 未选中态：白色背景，默认边框
- JS 交互：`selectRadioBtn(input)` 切换 is-active 状态，`getRadioVal(id)` 读取选中值
- 选项较多时（如转化目标 7 项、行动号召 10 项），radio-group 容器设置 `flex-wrap:wrap` 支持换行
```html
<div class="el-radio-group" id="campaignDeliveryStrategy">
  <label class="el-radio-button el-radio-button--small is-active">
    <input class="el-radio-button__original-radio" type="radio" name="..." value="2" checked onclick="selectRadioBtn(this)">
    <span class="el-radio-button__inner">目标成本</span>
  </label>
</div>
```
### 2.3.2 单选项静态展示
仅有一个可选项的字段（如广告分类=效果广告、营销目标=转化、计费方式=OCPM），显示为静态 `<span>` 文本而非 radio，避免无效交互。
### 2.3.3 hidden 属性强制规则
由于行内 `style="display:flex"` 会覆盖 HTML `hidden` 属性，全局增加 CSS 规则：
```css
[hidden]{display:none!important}
```
### 2.3.4 disabled 状态
不可操作的 radio 选项（如 CBO 联动限制的预算类型）采用 `opacity:0.4; pointer-events:none` 灰显禁用，保留视觉占位。
### 2.3.5 状态标签
- 成功: `.el-tag--success` (#67c23a)
- 警告: `.el-message--warning` (#e6a23c)
- 信息: `.el-message--info` (#909399)

## 2.4 账号配置卡片
### 2.4.1 卡片布局
横向滚动区第一张卡片（`min-width:280px`）：
- 头部：账号 + [清空] 按钮
- 内容区：已选账户标签（灰色 tag，含 ✕ 移除按钮）+ [新增] 按钮
- 底部无编辑按钮（操作直接在卡片上完成）
### 2.4.2 选择账号弹窗 (`dialogAccount`)
由卡片 [新增] 按钮触发，z-index 固定 5000：
- 搜索栏：搜索类型下拉（账户ID / 账户名称）+ 输入框 + [搜索] + [重置]
- 表格列：全选 checkbox、ID、账户名称
- 分页器：共 N 条 + 10条/页 + 页码
- 底部：[取消] [确定]
**确定行为**：
- 已存在的账户不重复添加（去重）
- 将选中账户渲染为 tag 添加到卡片的 `accountTags` 区域
- 关闭弹窗 → 更新预览区 → 提示成功
### 2.4.3 批量输入账号弹窗 (`dialogBatchInput`)
由选择账号弹窗中搜索输入框的搜索图标或回车键触发：
- 弹窗标题动态变化：根据搜索类型显示"请输入账户ID"或"请输入账户名称"
- textarea 区域，换行符分割
- [清空] 按钮清空 textarea
- [确认] 行为：按行解析输入 → 在 accountListData 中匹配（ID 精确匹配 `===`，名称模糊匹配 `indexOf`） → 替换 accountListData 渲染筛选结果 → 关闭弹窗提示

## 2.5 计划基本信息配置（子抽屉）
由卡片 [编辑] 按钮触发，打开 `drawer-plan`（width: 55%）。
### 2.5.1 广告计划设置
| 字段 | 控件类型 | API 参数 | 值/说明 |
|------|----------|----------|---------|
| 计划名称 | 文本输入 | `campaignName` | 最大 100 字符，必填 |
| 广告分类 | 静态文本 | `adCategory` | 1=效果广告（当前仅此一个值） |
| 营销目标 | 静态文本 | `campaignType` | 3=转化（当前仅此一个值） |
| 推广对象 | radio (2项) | `marketingType` | 1=应用推广 / 2=网站推广，切换时联动显隐应用选择区和落地页/Pixel配置 |
| 应用选择 | 下拉选择 | `appId` | 仅当 marketingType=1 时显示，当前可选 netshort |
| 智能投放 | radio (3项) | `autoPromotion` | 1=不开启 / 2=开启智能基建 / 3=智能基建+调控（默认 3） |
| 监测链接模版 | 只读输入 + [选择] 按钮 | — | 必填，点击按钮打开推广链接子抽屉 |
### 2.5.2 出价与预算
| 字段 | 控件类型 | API 参数 | 值/说明 |
|------|----------|----------|---------|
| 出价策略 | radio (2项) | `deliveryStrategy` | 3=最低转化成本 / 4=成本上限 |
| 预算优化CBO | radio (2项) | `cboAuthorized` | 0=关闭 / 1=开启（见 2.5.3 联动规则） |
| 预算类型 | radio (2项) | `budgetType` | 1=不限 / 2=每日固定预算 |
| 每日预算 | 文本输入 | `dayBudget` | 仅当 budgetType=2 时显示，USD。输入预期值，后端 `*10^6` 传输 |
### 2.5.3 CBO 联动规则
| CBO 值 | 预算类型可选值 | 说明 |
|--------|---------------|------|
| 关闭 (0) | 仅 1（不限） | 每日固定预算灰显禁用 |
| 开启 (1) | 仅 2（每日固定预算） | 不限灰显禁用 |
联动实现：`onCboChange()` → 禁用不可选项（opacity:0.4; pointer-events:none） → 自动切换到有效值 → 刷新显隐
### 2.5.4 保存行为 (`handleSavePlan`)
点击 [确定] 后：
- 如已保存过 (`_campaignSaved` 为 true)，弹确认框"保存计划将清空广告组、定向包和广告信息"
- 关闭子抽屉
- 将表单值回填到配置区横向卡片中的计划基本信息卡片
- 清空广告组卡片内容（重置为默认值）
- 清空定向包已选项（`selectedTargetingIds = []`）
- 清空创意已选项（`creativeAdgroups = []`，`_creativeSaved = false`）
- 设置 `_campaignSaved = true`
- 启用广告组/定向包/创意卡片的编辑按钮（`updateConfigButtons()`）
- 更新预览区表格
- 弹出"计划已保存，广告组配置已重置"提示
### 2.5.5 配置按钮锁定机制
- 初始状态：广告组、定向包、创意的编辑按钮灰显禁用（`color:#c0c4cc; cursor:not-allowed`）
- 计划保存后 (`_campaignSaved = true`)：启用全部编辑按钮
- 目的：确保必须先完成计划配置，再配置下游模块

## 2.6 推广链接管理（子抽屉）
由计划基本信息中监测链接模版的 [选择] 按钮触发，打开 `drawer-promo`（width: 960px）。
### 2.6.1 功能概述
- 表格列：选择(radio 单选)、ID、推广链接名称、投放策略、语言、操作(修改/复制/删除)

# 3. Kwai Marketing API 对接规范
## 3.1 接口层级
Account → Campaign (plan) → Unit (adgroup) → Creative
| 层级 | API 创建接口 | 批量上限 | 说明 |
|------|------------|---------|------|
| 计划 (Campaign) | `dspCampaignAddPerformance` | 1 | 单次创建 1 条 |
| 广告组 (Unit) | `dspUnitAddPerformance` | 1 | 单次创建 1 条 |
| 创意 (Creative) | `dspCreativeAddPerformance` | 15 | 单次最多 15 条，每个 Unit 最多 15 个创意 |
| 素材查询 | `dspMaterialPageQueryPerformance` | — | 分页查询已有素材 |
| 通用查询 | `postQuery` | — | 查询投放状态 |
## 3.2 通用参数
| 参数 | 说明 | 固定值 |
|------|------|--------|
| `dataGenerationSource` | 数据来源 | 2 (MAPI) — 所有 MAPI 创建的资源必须设为 2 |
## 3.3 金额单位
所有 API 中的货币值（`dayBudget`、`bid`、`budgetSchedule`）使用 `actual × 10^6` 的整数格式传输：
| UI 输入 (USD) | API 传输值 |
|---------------|-----------|
| 111 | 111000000 |
| 200 | 200000000 |
| 0 | 0 |
后端负责单位转换，前端 UI 展示和输入使用 USD 原始值。
## 3.4 API 错误处理
- 提交失败时，对应层级状态设为"提交失败"
- 结果列展示成功:N 失败:N 明细
- 失败项支持逐层重试（仅重试失败项，不影响已成功的）

# 全局技术备注
## 原型技术栈
- 单文件 HTML，使用 Element Plus 风格 CSS 变量
- 主色调: `#1677ff` (Ant Design 蓝调)
- 无外部依赖，纯 vanilla JavaScript
## 组件体系
### 抽屉系统
遮罩层 (Overlay) z-index = base + 0, 起始 2000, 每次 openDrawer 递增 2。抽屉 (Drawer) z-index = base + 1。
- `openDrawer(id)`: z-index 动态堆叠 → 添加 show class → 触发表单初始化钩子
- `closeDrawer(id)`: 移除 show class → 清除 z-index
- 弹窗 (Dialog) z-index 固定 5000，高于所有抽屉
- 弹窗遮罩层关闭：`event.target===this && closeDialog(id)`
### 弹窗系统
遮罩层 (DialogOverlay) z-index = 5000。弹窗 (Dialog) z-index = auto (在 overlay 内)。
- `openDialog(id)` / `closeDialog(id)` 控制 show class
### Radio Button
`selectRadioBtn(input)` → 清除同级所有 is-active → 给 input 所在 .el-radio-button 添加 is-active。`getRadioVal(groupId)` → 返回 group 内 checked radio 的 value。
### Toast
`showMsg(msg, type)` → 设置 el-message 文本和类型 → 添加 show class → 2s 后自动移除。支持类型: success / warning / info。
## 表单规范
- 所有配置类选择控件使用 radio button tabs，不使用 `<select>` 下拉框（例外：下拉选择如应用选择、素材库搜索条件、手机品牌等仍使用原生 select）
- 仅有一个可选项的字段显示为静态文本
- 选项较多时容器设置 `flex-wrap:wrap` 支持换行
## 字段联动
| 联动函数 | 触发时机 | 联动逻辑 |
|----------|---------|---------|
| `onMarketingTypeChange()` | 切换推广对象 | 应用推广显示应用选择区，隐藏落地页/Pixel；网站推广反之 |
| `onCboChange()` | 切换 CBO | CBO 关闭→计划预算只能不限；开启→只能每日固定预算；联动更新广告组预算可选项 |
| `onBudgetTypeChange()` | 切换计划预算类型 | 每日固定预算显示输入框 |
| `onUnitBudgetTypeChange()` | 切换广告组预算类型 / 切换 CBO | 每日显示输入框，固定日期显示分日预算，同时联动 CBO 限制 |
| `onOcpxChange()` | 切换转化目标 | ROAS 类型显示 ROI 出价(%)，非 ROAS 显示目标出价(USD) |
| `onScheduleTypeChange()` | 切换排期类型 | 自定义显示日期输入区 |
| `onTimeScheduleTypeChange()` | 切换投放时段类型 | 特定时间显示时段网格 |
## 全局 CSS 规则
```css
[hidden]{display:none!important}
```
- 解决行内 `style="display:flex"` 覆盖 HTML `hidden` 属性的问题
- `handleReset()` 仅清空搜索输入框，不清空下拉选择
## 安全函数
```js
function escHtml(s) { var div = document.createElement('div'); div.appendChild(document.createTextNode(s)); return div.innerHTML; }
function escJs(s) { return s.replace(/\\/g, '\\\\').replace(/'/g, "\\'").replace(/"/g, '\\"'); }
```
- `escHtml`: 渲染模板数据到 HTML 时防 XSS
- `escJs`: 模板名称嵌入 JS 字符串时防注入
