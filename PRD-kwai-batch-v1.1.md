# Kwai 批量创编系统 V1.1 PRD

## Problem Statement

V1.0 的创意配置流程要求运营逐个手动添加广告组、再逐个为每组选素材。当素材量达到几十甚至上百个时（例如选了 100 个视频素材，按每 3 个一组需要手动创建 34 个广告组并分 34 次选材），操作极其繁琐、效率极低。

## Solution

在创意抽屉中新增「批量创建作品」能力：运营设置"每组素材数"，一次性从素材库勾选所有素材，系统自动按顺序将素材分配到广告组中。把 34 次操作变成 1 次。

同时新增「多账户分配规则」，控制计划层级按何种维度展开（全量分配 / 按创意分配）。

## User Stories

1. 作为投放运营，我想在创意抽屉顶部设置"每组 N 个素材"，使系统按此规则自动分组
2. 作为投放运营，我想一次性从素材库勾选数十个素材，点确定后自动创建对应数量的广告组
3. 作为投放运营，我希望素材按在素材库中的勾选/展示顺序依次分配到各组（先选先进第一组），保持可预测性
4. 作为投放运营，我希望自动分组后已有的广告组不受影响，新组追加在末尾
5. 作为投放运营，我希望自动分组后仍能手动删除某个不满意的广告组或组内素材，拥有最终调整权
6. 作为投放运营，我希望广告组名称沿用广告组配置中的名称前缀 + 自动递增序号，保持命名一致性
7. 作为投放运营，我希望批量创建作品时固定走素材库（创建作品类型），不受发布身份设置影响
8. 作为投放运营，我希望"每组素材数"上限为 10，对应 Kwai 平台单个广告组的创意数量限制
9. 作为投放运营，我想在创意配置中选择多账户分配规则（全量分配 / 按创意分配），控制批量提交时计划与广告组的层级展开方式
10. 作为投放运营，我希望鼠标悬停到"多账户分配规则"标题上时看到两条规则的公式说明

## Implementation Decisions

### 批量创建作品

- **每组 N 个**：在创意抽屉顶部设置，全局生效，默认值 3，上限 10（Kwai 广告组创意数限制）
- **分配顺序**：按素材在列表中的展示顺序依次分配（ID 排序），不做随机或属性打散
- **追加模式**：自动分组结果追加到现有广告组列表末尾，不清空已有分组
- **素材来源**：固定走素材库弹窗（dialogMaterial），生成广告组类型固定为「创建作品」

### 多账户分配规则

- 在创意配置卡片下方，以 radio 形式提供两个选项：全量分配（默认）、按创意分配
- 提示文案放置在卡片标题"多账户分配规则"上，hover 显示规则公式
- 全量分配：计划数 = 定向包数，一个计划下的广告组 = 创意包，广告数 = 一个创意包素材数
- 按创意分配：计划数 = 创意包 × 定向包，一个计划下一个广告组，广告数 = 一个创意包素材数

### 命名规则

- 广告组名称 = 广告组配置中的名称前缀 + `-` + 3 位递增序号（如 `unitAddName-004`）
- 序号全局递增，不与已有广告组冲突

### 灵活性保留

- 自动分组后，每个广告组和组内素材均支持手动删除
- 原有"手动添加广告组 → 逐个选材"的流程保持不变，两种方式可混用
- 不支持拖拽/移动素材到其他组（V1.1 scope）

## Testing Decisions

- 本项目为产品原型，无自动化测试。验证方式：浏览器打开 `index.html`，手动执行以下场景：
  1. 默认 3 个/组，选 10 个素材 → 4 组（3+3+3+1）
  2. 修改为 5 个/组，选 10 个 → 2 组
  3. 设为 15 个/组 → 输入框上限 10，无法超出
  4. 已有 2 个手动广告组，再批量创建 6 个素材（每组 3）→ 追加 2 组，共 4 组
  5. 批量创建后点击组内素材 × 删除 → 正常移除
  6. 批量创建后点击广告组删除 → 正常移除整个组
  7. 切换多账户分配规则 → radio 切换正常，hover 标题显示规则说明

## Out of Scope

- 拖拽/移动素材到其他广告组
- 「总共 N 组」模式（只做"每组 N 个"）
- 素材按属性均匀打散
- 模板系统（V1.1 整个砍掉）
- 定时提交
- 多账户差异化素材配置
- 批量编辑广告组名称
- 按标题包分配（标题包概念暂不引入）
- 预览表按分配规则实时联动（后续迭代）
