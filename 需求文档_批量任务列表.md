# Kwai 批量创编系统 — 需求文档

## 系统概述

Kwai 批量创编系统是用于批量创建和管理 Kwai（快手海外）广告投放任务的 Web 工具。系统包含两大模块：

- **模块一：批量任务列表** — 展示、查询和管理批量投放任务，监控各层级提交状态
- **模块二：配置区** — 创建新投放任务，配置账号、模板、计划、推广链接、广告组等参数

**页面入口**: 投放系统 > Kwai投放

---

# 模块一：批量任务列表

## 1.1 模块概述

批量任务列表是系统的默认主页面，用于展示每一批次中各层级（计划 → 广告组 → 广告）的提交状态和结果，支持筛选、重试、删除等操作。

## 1.4 数据表格

### 1.4.1 表格列结构

表格采用**双层表头**设计，将计划、广告组、广告三层级的提交状态分别展示。

**第一层表头**（分组行）:

| 计划ID | 任务名称 | 账号名称 | 计划 (colspan=3) | 广告组 (colspan=3) | 广告 (colspan=3) | 创建时间 (东八时区) | 定时时间 | 操作 |
|--------|----------|----------|-------------------|---------------------|-------------------|---------------------|----------|------|

**第二层表头**（子列行）:

| → | → | → | 提交 | 状态 | 结果 | 提交 | 状态 | 结果 | 提交 | 状态 | 结果 | → | → | → |

- 任务名称列宽为 **8%**，支持折行
- 账号名称列宽为 **8%**，支持折行
- 其他列自适应宽度
- 创建时间列不折行 (white-space: nowrap)

### 1.4.2 列定义

| 列名 | 数据字段 | 说明 |
|------|----------|------|
| 计划ID | `planId` | 批量任务的唯一标识，由系统自动生成（如 893539） |
| 任务名称 | `taskName` | 用户自定义的任务名称，通常对应广告系列名称 |
| 账号名称 | `accountName` | Kwai 广告账户标识（如 FB-MAD-ZQIAAYY-UTC-0-01(1208877190771065)） |
| 计划-提交 | `planSubmit` | 计划层级提交数量（如 "1" 表示提交 1 个计划） |
| 计划-状态 | `planStatus` | 计划层级提交状态标签（见 1.4.3） |
| 计划-结果 | `planResult` | 计划层级提交结果明细（格式: "成功:N 失败:N"） |
| 广告组-提交 | `unitSubmit` | 广告组层级提交数量 |
| 广告组-状态 | `unitStatus` | 广告组层级提交状态标签 |
| 广告组-结果 | `unitResult` | 广告组层级提交结果明细 |
| 广告-提交 | `creativeSubmit` | 广告/创意层级提交数量 |
| 广告-状态 | `creativeStatus` | 广告/创意层级提交状态标签 |
| 广告-结果 | `creativeResult` | 广告/创意层级提交结果明细 |
| 创建时间 | `createTime` | 任务创建时间，东八时区 (UTC+8)，格式 `YYYY-MM-DD HH:mm:ss` |
| 定时时间 | `scheduleTime` | 定时提交时间，`-` 表示立即提交 |
| 操作 | — | 操作按钮（见 1.4.5） |

### 1.4.3 状态标签

| 状态值 | 标签样式 | 颜色 | 说明 |
|--------|----------|------|------|
| 提交完成 | 绿色标签 (`el-tag--success`) | #67c23a | 本轮提交已结束 |
| 等待提交 | 无特殊标签（灰色边框默认样式） | — | 尚未提交或重试后重置 |
| 提交失败 | 红色标签 (`el-tag--danger`) | #f56c6c | 提交过程中出现错误 |

### 1.4.4 结果列

- 格式: `成功:N 失败:N`
- 当 `失败:N` 中 **N > 0** 时，结果文本后追加 **重试** 按钮（蓝色链接样式，padding:0，font-size:11px）
- 点击重试后: 该层级状态重置为"等待提交"，结果重置为 `成功:0 失败:0`
- **重试按钮位于结果列**，不在状态列

### 1.4.5 操作列

每行包含两个操作按钮：

| 按钮 | 样式 | 行为 |
|------|------|------|
| 详情 | 蓝色链接按钮 | 打开配置区抽屉，查看/编辑该任务的完整配置 |
| 删除 | 红色链接按钮 | 从列表中删除该任务，弹出"已删除"提示 |

### 1.4.6 表格交互

- 行悬停高亮（浅灰背景）
- 奇偶行交替底色（斑马纹，通过 `.el-table__row--striped` 实现）

## 1.8 数据流

### 1.8.1 任务生命周期

```
[批量添加] → 配置区 → [立即提交/定时提交]
                       ↓
                  创建任务记录
                  (planStatus = "等待提交")
                       ↓
              Kwai API 批量调用
                       ↓
         ┌─────────────┼─────────────┐
         ↓             ↓             ↓
      计划提交      广告组提交     广告提交
         ↓             ↓             ↓
   成功/失败      成功/失败      成功/失败
         ↓             ↓             ↓
   planStatus     unitStatus    creativeStatus
   = "提交完成"   = "提交完成"   = "提交完成"
                       ↓
              [重试] → 失败项恢复为"等待提交"
```

### 1.8.2 重试机制

- 仅对**单个层级**（计划/广告组/广告）的失败项重试
- 重试后该层级状态立即变为"等待提交"，结果清零
- 不影响其他层级和其他任务

## 2.3 表单控件规范

### 2.3.1 Radio Button Tabs（选项卡切换）

**所有配置抽屉中的选择类字段统一使用 radio button tabs 代替下拉框**，样式特征：

- 使用 `.el-radio-group` + `.el-radio-button--small` 组合
- 选中态：蓝色背景 (`--el-color-primary`)，白色文字
- 未选中态：白色背景，默认边框
- JS 交互：`selectRadioBtn(input)` 切换 is-active 状态，`getRadioVal(id)` 读取选中值
- 选项较多时（如转化目标 8 项、行动号召 10 项、操作系统 7 项），radio-group 容器设置 `flex-wrap:wrap` 支持换行

```html
<div class="el-radio-group" id="campaignDeliveryStrategy">
  <label class="el-radio-button el-radio-button--small is-active">
    <input class="el-radio-button__original-radio" type="radio" name="..." value="2" checked onclick="selectRadioBtn(this)">
    <span class="el-radio-button__inner">目标成本</span>
  </label>
  <!-- 更多选项... -->
</div>
```

### 2.3.2 单选项静态展示

仅有一个可选项的字段（如计费方式=OCPM、板式类型=视频、广告分类=效果广告），显示为静态文本而非 radio，避免无效交互。同时保留 radio 结构方便后续扩展。

### 2.3.3 hidden 属性强制规则

由于行内 `style="display:flex"` 会覆盖 HTML `hidden` 属性，全局增加 CSS 规则：

```css
[hidden]{display:none!important}
```

### 2.3.4 状态标签

沿用 Element Plus 风格：
- 成功: `.el-tag--success` (#67c23a)
- 警告: `.el-message--warning` (#e6a23c)
- 信息: `.el-message--info` (#909399)

## 2.5 计划基本信息配置（子抽屉）

由卡片 [编辑] 按钮触发，打开 `drawer-plan`（width: 55%）。

### 2.5.1 广告计划设置

| 字段 | 控件类型 | API 参数 | 值/说明 |
|------|----------|----------|---------|
| 计划名称 | 文本输入 | `campaignName` | 最大 100 字符，必填 |
| 广告分类 | radio | `adCategory` | 1=效果广告（当前仅此一个值，预留扩展） |
| 营销目标 | radio | `campaignType` | 3=转化（当前仅此一个值，预留扩展） |
| 推广对象 | radio | `marketingType` | 1=应用推广 / 2=网站推广 |
| 智能投放 | radio | `autoPromotion` | 1=不开启 / 2=开启智能基建 / 3=智能基建+调控（默认 1，UI 默认选 3） |
| 监测链接模版 | 只读输入 + [选择] 按钮 | — | 必填，点击按钮打开"选择推广链接"子抽屉 |

### 2.5.2 出价与预算

| 字段 | 控件类型 | API 参数 | 值/说明 |
|------|----------|----------|---------|
| 出价策略 | radio | `deliveryStrategy` | 2=目标成本 / 3=最低转化成本 / 4=成本上限 |
| 预算优化CBO | radio | `cboAuthorized` | 0=关闭 / 1=开启（见 2.5.3 联动规则） |
| 预算类型 | radio | `budgetType` | 1=不限 / 2=每日固定预算 / 3=固定日期预算 |
| 每日预算 | 文本输入 | `dayBudget` | 仅当 budgetType=2 时显示，USD |
| 分日预算 | 7 个文本输入 | `budgetSchedule` | 周一~周日，仅当 budgetType=3 时显示。0 表示当日不限 |

分日预算示例值（对应 API `budgetSchedule` 的 `long[]`）：
- 周一: 400, 周二: 3000, 周三: 2000, 周四~周日: 0（不限）

> API 口径：`dayBudget` 和 `budgetSchedule` 均需将预期值 `*10^6` 传输。例如预期值 111 → 传输 `111000000`。UI 输入框显示的是预期值（USD），后端负责单位转换。

### 2.5.3 出价策略联动规则

| 出价策略 | CBO 行 | 预算类型可选值 |
|----------|--------|---------------|
| 目标成本 (2) | **隐藏**（Target Cost 不能填 cboAuthorized） | 1, 2, 3 全部可选 |
| 最低转化成本 (3) | **显示**（必填） | 仅 1（不限），2、3 灰显禁用 |
| 成本上限 (4) | **显示**（必填） | 仅 1（不限），2、3 灰显禁用 |

联动实现：
- `onStrategyChange()`: 策略切换时，更新 CBO 行显隐 + 预算类型可选项 + 自动切换到有效值
- `onBudgetTypeChange()`: 预算类型切换时，显示/隐藏每日预算输入框和分日预算输入区

```js
function onStrategyChange() {
  var strategy = getRadioVal('campaignDeliveryStrategy');
  // CBO: Target Cost(2) 隐藏, Lowest Cost(3)/Cost Cap(4) 显示
  document.getElementById('campaignCboRow').hidden = strategy === '2';
  // 预算类型: 3 或 4 时限定只能选"不限"
  var btns = document.querySelectorAll('#campaignBudgetType .budget-opt');
  btns.forEach(function(btn) {
    if (strategy === '3' || strategy === '4') {
      if (btn.querySelector('input').value !== '1') {
        btn.style.opacity = '0.4';
        btn.style.pointerEvents = 'none';
        // 如果当前选中了被禁用的项，自动切到"不限"
      }
    }
    // ...
  });
}
```

# 全局技术备注

- 原型 HTML 单文件实现，使用 Element Plus 风格 CSS 变量
- 主色调: `#1677ff` (Ant Design 蓝)
- 抽屉组件使用 CSS transform 动画 + z-index 动态堆叠（`openDrawer` 递增 2，起始 2000）
- 弹窗 (Dialog) z-index 固定 5000（高于所有抽屉），批量输入弹窗复用同一套 `openDialog`/`closeDialog` 机制
- 弹窗遮罩层点击关闭: `event.target===this&&closeDialog(id)`
- `handleReset()` 仅清空搜索输入框，不清空下拉选择
- `[hidden]{display:none!important}` 全局规则，解决行内 display:flex 覆盖 hidden 属性的问题

## 表单规范

- **所有配置类选择控件使用 radio button tabs**（`.el-radio-group` + `.el-radio-button--small`），不使用 `<select>` 下拉框
- Radio 切换通过 `selectRadioBtn(input)` 处理 is-active 状态
- Radio 取值通过 `getRadioVal(groupId)` 读取 `input[type=radio]:checked`
- 选项较多时（≥7 项），容器设置 `flex-wrap:wrap` 支持换行
- 仅有一个可选项的字段显示为静态文本 `span`，不渲染为 radio 或 select

## 字段联动

- 出价策略切换 → CBO 行显隐 + 预算类型可选项限制（`onStrategyChange()`）
- 预算类型切换 → 每日预算/分日预算区域显隐（`onBudgetTypeChange()`）
- 推广链接选择 → 校验投放策略与推广对象匹配关系（`confirmPromoSelect()`）

## 选择账号弹窗

- 账户选择弹窗中的"确定"操作包含去重逻辑：已存在的账户不会重复添加
- 批量输入弹窗中 ID 匹配为**精确匹配**（`===`），名称为**模糊匹配**（`indexOf`）
- 搜索结果和批量筛选结果通过临时替换 `accountListData` 渲染，完成后恢复原始数据
